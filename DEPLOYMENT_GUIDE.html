Overview

This document describes the steps required to update the live Shobhasiri dental website after making changes to any local files (e.g., index.html, style.css, images). The website is hosted on GitHub Pages and is automatically published whenever changes are pushed to the main branch on GitHub.

Tools Required

ToolPurpose
VS Code (or any text editor)Edit website files locally
Terminal / Command PromptRun Git commands to push changes
Web BrowserVerify the live site after deployment
GitHub AccountHosts the code and serves the website

Step-by-Step Procedure

1Make your changes locally

Open the project folder in VS Code and edit the files you want to change. Common files:

FileWhat it controls
index.htmlAll page content (text, sections, images)
style.cssColors, fonts, layout, and spacing
images/ folderPhotos and logos used on the site

Save the file after making changes (Ctrl + S).

2Preview changes locally (optional but recommended)

Open index.html directly in your browser to review your changes before pushing. Right-click the file → Open with → Browser.

Note: Local preview may look slightly different from the live site in rare cases, but it is good enough for checking text and layout changes.

3Open the Terminal in VS Code

In VS Code, go to Terminal → New Terminal (or press Ctrl + `).

Make sure you are in the correct project folder. The terminal prompt should show:

c:\Users\drsre\shobhasiri-website

If not, navigate there with:

cd "c:/Users/drsre/shobhasiri-website"

4Stage your changes

Tell Git which files you want to include in the update. To include all changed files:

git add .

Or to add a specific file only:

git add index.html

5Commit your changes

Save a snapshot of your changes with a short description of what you changed:

git commit -m "Describe what you changed here"

Examples of good commit messages:

git commit -m "Update doctor bio section" git commit -m "Add new service: teeth whitening" git commit -m "Fix contact phone number"

6Push to GitHub (deploy to live site)

git push origin main

This uploads your changes to GitHub. GitHub Pages will automatically detect the update and redeploy the live website.

7Wait for deployment

GitHub Pages usually takes 1 to 2 minutes to rebuild and publish the site.

To monitor deployment progress:

  1. Go to https://github.com/madala9999/shobhasiri-website
  2. Click the Actions tab
  3. You will see a workflow called "pages build and deployment" running
  4. A green checkmark means the deployment is complete

8Verify the live site

Open your browser and visit:

https://madala9999.github.io/shobhasiri-website/
Tip: If you don't see your changes, press Ctrl + Shift + R (hard refresh) to clear your browser cache and reload the latest version.

Quick Reference – Terminal Commands

CommandWhat it does
git statusShows which files have been changed
git add .Stages all changed files for commit
git commit -m "message"Saves a snapshot with a description
git push origin mainPushes changes to GitHub (goes live)
git log --onelineShows history of past commits

Troubleshooting

ProblemSolution
Changes not showing on live site Wait 2 minutes, then do a hard refresh (Ctrl + Shift + R)
"Nothing to commit" message The file was not saved, or the changes were already pushed. Check with git status
Push rejected / authentication error Make sure you are logged into GitHub in VS Code and have permission to push
Site shows old content after push Check the Actions tab on GitHub to confirm deployment completed successfully
